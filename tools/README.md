# tools/

Scripts and source data used to (re)generate the palette files.

## `build_palettes.py`

Reads each JSON file in `primitives_json/`, runs each through the
appropriate palette-builder function (matching the upstream Lua
`generate_spec()` in each `palette/*.lua`), eagerly computes all blends,
and emits a `.vim` file per primitive and per theme variant.

**Run:**

```sh
python3 tools/build_palettes.py
```

**Output paths** are hardcoded near the top of the script (`OUT_BASE`).
Update if you've moved the project.

## `primitives_json/`

The primitives data extracted from upstream's
`lua/github-theme/palette/primitives/*.lua` files. Those Lua files embed
JSON inside `[=[ ... ]=]` long-string brackets — `build_palettes.py`
doesn't know how to parse Lua, so the JSON was extracted upfront with a
small regex pass.

If upstream updates their primitives, re-extract by:

1. Cloning a fresh `github-nvim-theme`
2. For each `lua/github-theme/palette/primitives/<n>.lua`, pull out the
   JSON between `[=[` and `]=]` markers
3. Save as `primitives_json/<n>.json`
4. Re-run `build_palettes.py`

## What's NOT here

- The companion themes (Terminal.app, Xcode, bat) were generated by
  one-shot inline scripts during the original development session. If
  you need to regenerate them or produce them for other variants, the
  palette data is already in `autoload/github_theme/palette/*.vim` —
  just write a small script that loads the relevant colors and emits
  the appropriate output format. See `CLAUDE.md` "Companion themes" for
  the structures of each format.
